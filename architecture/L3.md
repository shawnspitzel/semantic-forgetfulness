# L3: Long-Term Gist Store

> L3 is the deepest tier of the Semantic Forgetfulness memory hierarchy. It stores aggressively compressed segment representations — concept embeddings in the LLM's own token embedding space — and surfaces them on demand via semantic retrieval. It is the tier that makes forgetting recoverable.

---

## What L3 Stores

Each L3 entry corresponds to one compressed context segment (≈20 tokens, sentence-boundary aligned):

```
L3Entry {
  id:                    UUID
  concept_embeddings:    Tensor[C, D]   # the gist — C concept embeddings of dim D
  representative_vec:    Tensor[D]      # mean-pool of CEs, used for ANN lookup
  sanity_anchors: {
    boundary_sentences:  [str, str]     # first + last sentence of original segment
    entities:            [str, ...]     # NER output (names, numbers, dates)
    semantic_fingerprint: Tensor[768]   # frozen MiniLM encoder output
  }
  metadata: {
    importance_score:    float          # attention concentration at encoding time
    encoded_at:          timestamp
    last_accessed:       timestamp
    access_count:        int
    source_position:     int            # original token position in conversation
    session_id:          str
  }
}
```

The `concept_embeddings` tensor is the primary artifact. It lives in the base LLM's token embedding space and can be prepended directly to any forward pass without decoding to text first (following CompLLM's design). It is **not** a generic semantic vector — it is an LLM-native compressed representation.

---

## Storage Architecture

**Everything lives in CPU DRAM. Nothing touches disk during a session.**

The disk-I/O latency penalty (>100ms from layout transformation overhead alone, per tiered KV cache literature) puts any disk-backed store outside our L3 budget. The full L3 index is memory-resident.

Two co-located structures:

### 1. HNSW Vector Index
An HNSW (Hierarchical Navigable Small World) index over the `representative_vec` of each entry, backed by `hnswlib` or FAISS.

- Query: a mean-pooled CE vector derived from the current context
- Returns: top-k candidate L3 entries by cosine similarity
- Expected latency: ~10ms at 100K entries
- Index is updated incrementally as segments are evicted from L2

### 2. In-Memory Entity Graph
A lightweight graph (NetworkX or custom adjacency list) over named entities and concept anchors extracted at compression time.

- **Nodes**: named entities, dates, and key noun phrases from `sanity_anchors.entities`
- **Edges**: co-occurrence within the same segment, or embedding cosine similarity above threshold τ (built at write time)
- **Purpose**: 1-hop BFS expansion after the ANN hit — if a retrieved segment mentions "attention mechanisms," adjacent graph nodes pull in semantically related segments without a second ANN query

Graph linking is built **at write time** (when segments are compressed and stored), keeping read-time overhead negligible (<1ms for 1-hop expansion). This asymmetry — expensive offline indexing, cheap online lookup — is the key design principle (following A-MEM's pattern).

## Retrieval

### Trigger: Demand-Paging (Fault-Driven)
L3 is queried only on an **L2 cache miss** — when the system determines that needed context is not present in L1 or L2. L3 is never proactively scanned. This is the demand-paging model from Pichay et al. 2026.

### Retrieval Pipeline

```
1. L2 miss detected
        ↓
2. Encode current query/context → mean-pooled CE query vector
        ↓
3. ANN search (HNSW) → top-k candidate segments            ~10ms
        ↓
4. Entity graph expansion (1-hop BFS) → expanded candidate set  <1ms
        ↓
5. [Optional] Rerank by full CE sequence cosine similarity   ~5ms
        ↓
6. Sanity check: cosine(reconstructed, semantic_fingerprint) ≥ θ
        ↓
7. Transfer CE tensor to GPU → prepend to LLM context       ~1ms
        ↓
8. Promote entry to L2; log access for training signal
```

**Total L3 budget: ~20–25ms.** Leaves 75ms headroom within our 100ms target.

## GNN Extension (Lab Integration)

The entity graph is a direct substrate for a learned GNN retriever, leveraging the lab's foundational model expertise.

Instead of fixed BFS expansion, a lightweight GNN trained on the entity co-occurrence graph learns to predict **retrieval relevance** given a query embedding — performing message-passing over the entity graph to score which connected segments are most useful to surface.

This replaces the heuristic 1-hop expansion with a learned, differentiable routing step. It is a potential second contribution in the paper and connects the project naturally to the lab's research program.

## Eviction

When L3 capacity is exceeded, entries are evicted by:

```
eviction_score = 0.6 * importance_score + 0.4 * exp(-λ * (t_now - t_last_accessed))
evict argmin_s eviction_score(s)
```

Evicted L3 entries are **dropped permanently** for the session. L3 is the floor of the hierarchy.

---

## What L3 Is Not

- **Not a vector database** in the RAG sense — retrieval is demand-paged (miss-driven), not query-driven. L3 is not searched on every turn.
- **Not text storage** — entries are concept embedding tensors, not raw or summarized text. There is no decode-to-text step on retrieval.
- **Not cross-session** — L3 is scoped to a single session. Cross-session persistence is explicitly out of scope for the initial research contribution (see Pichay et al. 2026 for framing).
- **Not disk-backed** — no SQLite, Neo4j, or NVMe during a session. All in DRAM.

## Key Design Decisions & Rationale

| Decision | Choice | Rationale |
|---|---|---|
| Storage backend | In-memory HNSW + entity graph | Only path to <100ms L3 budget |
| CE format | Dense float16 tensor in LLM embedding space | Directly injectable as soft tokens; no decode step |
| Graph structure | In-memory entity co-occurrence graph | Enables associative retrieval; GNN-extensible; build at write time |
| Retrieval trigger | L2 cache miss (demand-paging) | Avoids unnecessary L3 scans; mirrors OS virtual memory model |
| Retrieval mechanism | ANN search + 1-hop graph expansion | Fast (HNSW ~10ms) + associative recall that ANN alone cannot provide |
| Sanity check | Frozen MiniLM cosine similarity ≥ θ | Catches hallucinated reconstructions before L1 promotion |
| Eviction | Importance-weighted LRU | Aligns eviction policy with training objective (cache-miss signal) |
